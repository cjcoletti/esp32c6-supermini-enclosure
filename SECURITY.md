# Security Policy

## Supported versions

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |

Older versions are not maintained.

## Scope

This project provides **lab / hobby 3D-printable enclosure files** (OpenSCAD source and STL exports) for ESP32-C6 Super Mini modules. It is **not** a certified commercial product or an electrical safety enclosure. Reports about missing certifications, expected lab-only behavior, or general 3D-printing tolerances are out of scope for security advisories.

## Reporting a vulnerability

If you believe you have found a **security vulnerability** in this repository (for example: malicious content embedded in repository files, or a supply-chain issue affecting distributed assets), please report it **privately**.

**Do not** open a public GitHub Issue for security reports.

### Preferred: GitHub Security Advisories

Use [Report a vulnerability](https://github.com/cjcoletti/esp32c6-supermini-enclosure/security/advisories/new) on this repository. GitHub allows private disclosure and coordinated discussion before any public fix.

### What to include

- Description of the issue and potential impact
- Steps to reproduce, if applicable
- Affected release (e.g. `v1.0.0`) and, if known, parametric model (e.g. `V16.3`)
- Your environment (printer, slicer, board/PSU variant) when relevant

## Response expectations

This is a **personal open-source project**. There is no official SLA, but I aim to acknowledge reports within a reasonable time and to patch confirmed issues when feasible.

Thank you for helping keep the project safer for the community.
